## Objective

Implement three grammar activities directly aligned to CE Question 3: case identification, verb parsing, and paradigm self-check. These activities are generated from the existing grammar JSON — no new data files needed.

## Background

CE Question 3 asks pupils to:
- Identify the case of a noun ("What case is 'puellam'?")
- Identify the function of a case ("Why is this case used?")
- Parse a verb: person, number, tense ("What is the person and tense of 'portābam'?")

All questions derive from the existing `data/grammar/nouns.json` and `data/grammar/verbs.json`.

## Task 1: Generate case identification questions from nouns.json

**Files:** `js/exercises.js` (extend)

**Action:**
Add a `buildCaseQuestions()` function that loads `data/grammar/nouns.json` and generates one question per case/number combination for each declension. Each question:
- Displays a noun form (e.g. "puellam")
- Asks for the case (4 MCQ options)
- After answering, shows which case it is AND the function of that case

Case function explanations (embed in JS, not a separate file):
```javascript
const CASE_FUNCTIONS = {
  nominative: 'Used for the subject of the verb — the person or thing doing the action.',
  vocative:   'Used when directly addressing someone.',
  accusative: 'Used for the direct object — the person or thing the action is done to.',
  genitive:   'Used to show possession or relationship ("of").',
  dative:     'Used for the indirect object — the person something is given to or done for.',
  ablative:   'Used after many prepositions (in, cum, ex, ab, de, sub) and to show means or manner.'
};
```

Add `runCaseIdentifier(app, nounData)` function:
- Load `data/grammar/nouns.json` via fetch
- Flatten into question array: for each declension × case × number, one question with: `{ form, case, number, declension, declensionName }`
- Shuffle and take 10
- Render MCQ: show the form, ask "What case is this?" with 4 case options (from the 5 cases: nominative, vocative, accusative, genitive, dative, ablative — pick 3 wrong distractors)
- After answer: show ✓/✗, the correct case, and `CASE_FUNCTIONS[correctCase]`
- Label: "Practises: CE Question 3 (grammar — case identification)"

Add route in the activity router switch:
```javascript
case 'case': return runCaseIdentifier(app);
```

**Verify:**
- `quiz.html?activity=case` renders a question with 4 case option buttons
- Tapping the correct case shows green feedback with the case function explanation
- Tapping wrong shows the correct answer in feedback

**Done:** Case identifier activity functional, generating real questions from grammar JSON, with case function explanations after each answer.

## Task 2: Build verb parser activity from verbs.json

**Files:** `js/exercises.js` (extend)

**Action:**
Add `runVerbParser(app)` function. Load `data/grammar/verbs.json`. For each conjugation × tense × person × number, generate a question: { form, conjugation, tense, person, number }.

Verb form display: show the form (e.g. "portābam"), ask THREE sub-questions in sequence (or one combined MCQ):
- Person: 1st / 2nd / 3rd (MCQ, 3 options)
- Number: singular / plural (MCQ, 2 options)
- Tense: present / imperfect / perfect (MCQ, 3 options)

For simplicity: present all three as a single MCQ question with 4 fully-specified options, e.g.:
- Option A: 1st person singular, present
- Option B: 3rd person singular, imperfect ← correct
- Option C: 2nd person plural, present
- Option D: 1st person plural, perfect

After answering: show the full parsing ("portābam: 1st person singular imperfect of portō — 'I was carrying'") and a note: "The imperfect tense describes a continuous or repeated action in the past."

Tense explanations (embed in JS):
```javascript
const TENSE_NOTES = {
  present:   'The present tense describes an action happening now or habitually.',
  imperfect: 'The imperfect tense describes a continuous or repeated action in the past.',
  perfect:   'The perfect tense describes a completed action in the past.'
};
```

Add route: `case 'verb': return runVerbParser(app);`

**Verify:**
- `quiz.html?activity=verb` shows a Latin verb form with 4 fully-specified option buttons
- Correct answer shows the full parsing and tense note
- Wrong answer shows the correct parsing

**Done:** Verb parser generates real questions from verbs.json with meaningful post-answer explanations.

## Task 3: Paradigm self-check mode

**Files:** `js/exercises.js` (extend)

**Action:**
Add `runParadigmCheck(app)` function. This shows a complete noun declension table (chosen randomly or by `?filter=decl:1`) with 4 randomly blanked cells. Pupil clicks a blanked cell, sees 4 options (the correct form + 3 other forms from the same paradigm), selects one.

Layout:
- Table displays exactly as in grammar.html (same CSS classes)
- Blanked cells show a "?" button
- On tap, a small option panel slides in beneath the table with 4 choices
- Correct: cell fills in with green highlight; Wrong: cell shows correct form in red, then corrects itself

After all 4 blanks filled: show summary ("4/4 correct — you've got this paradigm!") with "Try another" and "Back to Exercises" buttons.

Add route: `case 'paradigm': return runParadigmCheck(app);`

Also add "Paradigm Self-Check" to the hub card list in `renderHub()`:
```javascript
{
  id: 'paradigm',
  title: 'Paradigm Check',
  desc: 'Fill in the blanks in a noun or verb table.',
  ce: 'Practises: CE Question 3 & 4',
  href: 'quiz.html?activity=paradigm',
  icon: '📊'
}
```

**Verify:**
- `quiz.html?activity=paradigm` renders a noun paradigm table with 4 "?" buttons
- Tapping a "?" shows 4 form options
- Correct selection turns the cell green
- After all 4 filled, summary shown

**Done:** Paradigm check renders a real declension table from grammar JSON with interactive blanked cells.

## must_haves

- `quiz.html?activity=case` shows a Latin noun form and asks the student to identify the case from 4 options
- After answering a case question, the function of that case is explained in plain English (e.g. "Used for the subject of the verb")
- `quiz.html?activity=verb` shows a Latin verb form and asks for person/number/tense from 4 combined options
- After answering a verb question, the tense meaning is explained (present = now/habitually, imperfect = continuous past, perfect = completed past)
- `quiz.html?activity=paradigm` shows a noun table with 4 blanked cells the student can tap to fill in
- All three activities are labelled "Practises: CE Question 3 (grammar)"
- All three work fully offline (data comes from cached JSON)
